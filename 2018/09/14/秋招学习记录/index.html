发表于 2018-09-14

面试也是学习~回答不上的，都是我应该学习的

设计高并发服务器

  1. 负载均衡（nginx，ip_hash，url_hash，权重，轮询）
  2. 缓存（缓存同步的问题-不同步：通过负载均衡设置某类请求只能达到某个服务器；同步：互相发消息）
  3. 限流（流量不要一次性到达数据库，直接截断或者存入队列）
  4. 数据库的缓存（类似mybatis缓存，根据sql语句和返回结果，存入hashmap中，更新）
  5. 数据库架构设计（分库分表，读写分离-主从，优化数据表结构-字段的长度，主键，外键之类的）

net filter：一种，可以在应用层实现ip层钩子的东西，可以用来查看ip层具体发了那些包

java wait和sleep区别：sleep不释放锁，wait释放锁，wait和notify联用，必须在有锁的情况下使用（面试回答到锁层面，但是记反了）

字符串，找出第一个只出现一次的字符，两种思路，1、维持两个list，一个存只出现了一次的字符list1，一个存出现了多次的字符list2，遍历一次字符串，如果出现的字符没有在list2中，并且在list1中也没有，则加入list1，如果list1中存在，则将之放到list2；已经在list2中的，直接忽略跳过 2、动态规划，维持一个两维数组int[input.length][26]，遍历一次字符串，数组每一行int[i]，表示0-i的子字符串中每个字母出现的次数，遍历完成后，找出以后一行中第一个1所对应的字符就ok，思路是这样，因为是第一个出现的字符，所以还需要再加一部分代码。第二种方法更加全面，比如题改成第一个出现两次的字符….可惜我在面试的时候没说出来这方面~ 面试官似乎不是很满意我的解法

url如下：xxxx/contextPath(某个具体的webapp名字)/servletPath;

Spring+tomcat收到请求详细的行为情况：tomcat中，Connector收到请求（创建request和response对象），将请求交给Connector对应的Engine进行处理，Engine根据host找到具体的Host处理器，Host根据url中的contextPath找到具体Context（Context包含某个应用下的所有servlet，可以理解成那个应用），Context再根据servletPath找到对应的Wrapper（即具体的servlet），执行wrapper中的代码即可~，最后一层一层的返回

tomcat和Spring的理解：tomcat是servlet容器，Spring是java bean的容器；写完项目之后，将war包放入tomcat webapp目录下，tomcat会进行加载。

O(n)的排序方法：桶排序（空间消耗大）；数组中无序，但是值是1-n，可以使用角标交换法

大数据，找中位数：划分区域，然后读取数据，数据根据大小，放在某个特定区域中，计算所有区域总和n，从小区域到大区域数量相加，直到达到n/2，说明中位数处于当前区域，然后在该区域中寻找第x个数（方法与上述相同）

java中，多线程，其中一个线程崩掉，程序是否会崩掉？不会（单从线程和进程之间来讲，某个进程之中，一个线程崩掉，会导致整个进程崩掉，jvm中进行了特殊的处理（猜测是大的try catch包起来），线程崩掉不会影响其他线程）

数据库索引知识（具体到索引存储，加载到内存中，检索速度巴拉巴拉一堆）

mysql事务默认隔离级别-可重复读；使用mvcc解决幻读问题；注：可重复读隔离级别 不能解决幻读，mysql中使用mvcc达到解决幻读的问题

mysql 会对sql中where子句根据索引情况进行优化

linux中，查看共享内存情况 ipcs -m；查看端口占用（进程占用哪些套接字）netstat；查看端口被哪个进程占用 lsof；输出文本最后100行 且需要进行滚动（-f效果）：tail -f -n 100 xxx
